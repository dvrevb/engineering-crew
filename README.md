# Engineering Crew

A multi-agent AI pipeline that builds software from requirements. Describe what you want 
to build — the crew designs the architecture, plans execution, writes backend and frontend 
code to disk, and produces tests. All output lands in a named project folder (e.g. `TraderSim/`) 
derived from your requirements.

---

## How it works

Five agents run in sequence:

| # | Agent | What it does |
|---|---|---|
| 1 | **Engineering Lead** | Designs architecture, tech stack, folder tree, API contracts, task breakdown |
| 2 | **Engineering Manager** | Validates architecture, assigns filtered tasks per role |
| 3 | **Backend Engineer** | Implements backend and writes files to disk |
| 4 | **Frontend Engineer** | Implements a Gradio UI that calls the backend over HTTP |
| 5 | **Test Engineer** | Writes unit and integration tests |

Output is written to a project folder named after the application (e.g. `TraderSim/`). A run summary is also saved to `output/result.md`.

## Requirements

- Python `>= 3.10, < 3.14`
- [uv](https://docs.astral.sh/uv/)
- [CrewAI](https://crewai.com) — agent orchestration framework

## Installation

```bash
pip install uv
uv sync
```

---

## Environment variables

Create a `.env` file in the project root — never commit it.

```ini
OPENAI_API_KEY=your_openai_key         # engineering lead + manager
ANTHROPIC_API_KEY=your_anthropic_key   # backend, frontend, test engineers
```

Models are configured in `src/engineering_crew/config/agents.yaml`.

---

## Usage

**1. Set your requirements**

Edit the `requirements` string in `src/engineering_crew/main.py`:

```python
requirements = """
Your project description here.
"""
```

**2. Run the crew**

```bash
crewai run
# or
uv run engineering_crew
```

**3. Find your output**

```
output/result.md          ← crew run summary
<ProjectName>/            ← generated project folder
    main.py
    backend/
    frontend/
    tests/
```

---

## Customize the pipeline

| File | Purpose |
|---|---|
| `src/engineering_crew/config/agents.yaml` | Agent roles, models, delegation rules |
| `src/engineering_crew/config/tasks.yaml` | Task descriptions, expected outputs, context chains |
| `src/engineering_crew/crew.py` | Crew wiring, tools, process |
| `src/engineering_crew/main.py` | Entry point and requirements input |

## TraderSim — example output

`TraderSim/` is a reference implementation generated by Engineering Crew — a trading account simulation platform.

- **Backend** — FastAPI REST API with SQLite persistence
- **Frontend** — Gradio 4.x UI talking to `http://localhost:8000`
- **Tests** — pytest unit and integration tests

### Run TraderSim

Open two terminals from the project root:

**Terminal 1 — Backend**
```bash
cd TraderSim
uv run python main.py
```
API runs on `http://localhost:8000`. Interactive docs at `http://localhost:8000/docs`.

**Terminal 2 — Frontend**
```bash
cd TraderSim
uv run python frontend/app.py
```
Gradio UI opens at `http://localhost:7860`.

### Run tests

```bash
cd TraderSim
uv run pytest tests/ -v
```
